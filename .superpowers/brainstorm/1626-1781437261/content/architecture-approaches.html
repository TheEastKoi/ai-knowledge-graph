整体架构方案对比

现有项目是一个 D3.js 知识图谱，需要拆解为 Vue 组件化应用。以下是三种重构方向：

A

保留知识图谱 + 增强

核心仍为可视化知识图谱，但用 Vue 组件化拆分。增加搜索、过滤、节点收藏、学习进度追踪等增强功能。适合保持原有交互体验的升级。

B

多视图知识学习平台

图谱只是一个视图。增加：列表/卡片浏览、分类筛选、详情页（含代码示例）、学习路线图、面试题库、搜索等。更接近完整的知识学习 Web 应用。

C

极简文档型知识库

放弃图谱，转为文档/百科风格。左侧目录树，右侧 Markdown 详情。类似 VuePress/Docusaurus 风格。更注重内容深度，而非可视化交互。

UI 设计风格方向

无论选择哪种架构，都需要确定视觉风格

暗色仪表盘风格

保持现有暗色主题，向 Dashboard 方向进化。科技感强，适合技术类内容的展示。

极简编辑风格

浅色主题，大量留白，字体排版驱动。类似 Linear/Notion 风格，优雅克制。

赛博科技风

霓虹渐变、毛玻璃、发光边框。深色背景 + 彩色光效，面向开发者的炫酷风格。
